1. ユーザーがファイルパスを指定した場合 → そのパスを使用
2. 指定がない場合 → 以下のディレクトリから **最新の assetdata_*.csv** を使用
   ```
   /Users/matsubarakouji/ai-agent/economics/global-macro-strategy/data/assets/
   ```

**データ処理規則**:
- `NO` 列が空白の行（区切り行）は**除外**する
- Ticker が空白の行も**除外**する
- 同一ティッカーが複数行ある場合は**最初の出現のみ**使用する
- パフォーマンス列（`Per Yesterday`, `7`, `14`, `30`, `90`, `0101`, `365`）は**小数形式**のため、表示時は×100して%表示する
  - 例: `-0.0023` → `-0.23%`
- `Per Yesterday` が `-1.0` 以下、または価格が `0` や `1` の銘柄は**データ異常**として除外し、後述の品質注記に記載する

### 0-4. データ品質チェック

異常値（価格=0, 1、リターン=-100%超）を持つ銘柄を特定し、分析から除外する。

確認できなかった情報は「確認できず」と明記し、推論であることを「〜と考えられる」で明示する。

## STEP 2: アノマリー検知（データ分析）

読み込んだCSVデータを使い、以下の3種類のアノマリーを検知してください。

### 2-1. 相関ブレイクの検知

通常の相関関係から大きく乖離しているペアを特定する。

**監視すべき主要相関ペア**:
| ペア | 通常の関係 | ブレイクの意味 |
|---|---|---|
| SPY vs TLT | 逆相関（株↑国債↓）| 同方向 → 特殊なレジーム |
| SPY vs HYG | 正相関（リスクオン連動）| 乖離 → 信用市場が先行シグナル |
| GLD vs UUP | 逆相関（金↑ドル↓）| 同方向 → 安全資産需要の異常 |
| USO vs XLE | 強正相関 | 乖離 → セクター特有の要因 |
| QQQ vs XLU | 逆相関（リスクオン↔ディフェンシブ）| 同方向 → センチメント転換 |

**判定基準**: `Per Yesterday` の乖離が通常と逆方向かつ顕著（±1%以上の差）な場合

### 2-2. ボラティリティ・スパイクの検知

| ETF | アノマリー基準 |
|---|---|
| VIXY | `Per Yesterday` が +5% 以上 → 市場の恐怖急増 |
| 個別ETF | `Per Yesterday` の絶対値が `7日間の日次平均変動率` × 2倍以上 |

**日次平均変動率の計算**: `|7日間リターン| ÷ 7`

### 2-3. トレンド加速・反転の検知

`Per Yesterday` の方向が `7日間リターン` の方向と**逆転**しているETFを特定する。

- **ヘッダー（核心的テーマ）**: 当日の市場を動かした核心を1文で。データとWebリサーチ両方に基づくこと

- **付録D**: `Per Yesterday` 列で上位5・下位5を特定。Webリサーチで変動材料を確認
- **付録E**: 各アセットクラスは「概観→ドライバー→内部ダイナミクス→総括」の4要素で記述
