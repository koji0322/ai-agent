この分類は、我々の分析能力の現在地を正確に把握し、今後どのデータソースを強化すべきかを特定する上で極めて重要です。

### **フレームワーク分析項目のデータ要件分類**

**結論サマリー：**

*   **提供されたETFデータで可能なこと**: 主に、市場で**「何が（What）」**起きているかを詳細に分析することです。どのセクター、国、テーマが強く、どれが弱いかという**「市場のパフォーマンス地図」**の作成と、それに基づく**「ポートフォリオの具体的な構築」**において絶大な力を発揮します。

*   **追加の外部データが必要なこと**: 市場で**「なぜ（Why）」**それが起きているのかという背景要因の分析、そして市場の**「内部構造の健全性（How）」**の評価、さらに**「将来のリスク管理（Risk Management）」**において不可欠です。これには、経済指標、金融政策、市場のミクロ構造データなどが含まれます。

以下に、6つのピラーに沿った詳細な分類表を示します。

| 分析項目 (Analysis Item) | データ要件 (Data Requirement) | 補足 (Supplement) |
| :--- | :--- | :--- |
| **ピラー I：グローバルマクロ環境分析** | | |
| 1. ビジネスサイクル分析 | **外部データが必要** | ETFのセクター別パフォーマンス（例：XLE vs XLU）は**景気サイクルの示唆**を与えますが、GDP、失業率、PMIなどの経済指標がなければサイクル段階を特定できません。 |
| 2. 金融・財政政策分析 | **外部データが必要** | 中央銀行の政策金利、声明、議事録、政府の財政法案などのデータは、ETFデータには含まれていません。 |
| 3. 地政学リスク分析 | **外部データが必要** | ETFのパフォーマンス（例：ITAの上昇）はリスクの**結果**を示しますが、紛争や選挙といった**原因**の分析にはニュースや専門家のレポートが必要です。 |
| 4. 長期・構造的トレンド分析 | **外部データが必要** | AI革命や人口動態といったトレンドの**背景**を理解するには、技術レポートや統計データが不可欠です。ETFデータは、どのテーマが市場で評価されているかを示すに留まります。 |
| **ピラー II：クロスアセット・テーマ別市場分析** | | |
| 1. アセットクラス別分析 | **ETFデータのみで可能** | 株式(SPY)、債券(AGG)、コモディティ(DBC)など、各ETFのパフォーマンスを比較することで、リスクセンチメントを直接分析できます。 |
| 2. 株式市場の深掘り | **ETFデータのみで可能** | 国(EWC)、セクター(XLK)、テーマ(ARKK)、スタイル(VTV)など、提供されたデータはまさにこの分析のために最適化されています。パフォーマンス比較が容易です。 |
| **ピラー III：定量的・市場内部構造分析** | | |
| 1. 信用スプレッド分析 | **外部データが必要** | 各社債ETFの「価格」ではなく、その「利回り」と国債利回りの差（スプレッド）を計算する必要があるため、リアルタイムの利回りデータが必要です。 |
| 2. イールドカーブ分析 | **外部データが必要** | 各年限の国債利回り（2年、10年など）のデータが必要です。ETFの価格からでは正確なイールドカーブを描けません。 |
| 3. ボラティリティ構造分析 | **外部データが必要** | VIX先物カーブやオプション市場のデータ（プット・コールレシオ等）が必要です。VIXYの価格はあくまで短期先物の一点を示すに過ぎません。 |
| 4. 市場間レシオ分析 | **ETFデータのみで可能** | 各ETFの「価格」の比率を計算する（例：QQQの価格 ÷ SPYの価格）ことで、相対的な強弱を分析できます。 |
| 5. 資金フロー分析 | **外部データが必要** | 各ETFへの資金の純流入・流出額データは、専門のデータプロバイダーから入手する必要があります。 |
| **ピラー IV：シナリオ・プランニングと統合** | | |
| 1. 主要ドライバーの特定 | **両方が必要** | ETFデータから**パフォーマンスのドライバー**を特定し（What）、外部データ（ピラーI）でその**背景要因**（Why）を裏付け、統合します。 |
| 2. シナリオの構築 | **外部データが必要** | シナリオはマクロ経済の将来予測に基づきます。GDP成長率、インフレ率、政策金利などの将来予測データや専門家の見解が必要です。 |
| 3. サインポスト（先行指標）の設定 | **外部データが必要** | CPI発表値、FRB高官の発言、選挙結果など、シナリオの分岐点を判断する指標はすべて外部情報です。 |
| **ピラー V：戦略構築とポートフォリオ実践** | | |
| 1. 投資戦略フレームワークの選択 | **両方が必要** | ETFデータから市場の二極化（バーベル市場）を認識し、外部のマクロ分析と統合して、バーベル戦略のような最適なフレームワークを選択します。 |
| 2. ポートフォリオの構築 | **ETFデータのみで可能** | 戦略（例：トライデント戦略）を決定すれば、その構成要素となるETF（例：SMH, TLT, GLD）は提供されたリストから直接選択し、配分を決定できます。 |
| 3. ヘッジ戦略の導入 | **ETFデータのみで可能** | 市場全体をショートするETF（例：SH, PSQ）などをリストから選択することで、基本的なヘッジポジションを構築できます。 |
| **ピラー VI：リスク管理とプロセス改善** | | |
| 1. ポートフォリオのストレステスト | **外部データが必要** | 過去の危機（リーマンショック等）でテストするには、提供データより長期のヒストリカル価格データが必要です。 |
| 2. 流動性リスク管理 | **外部データが必要** | 各ETFの平均日次出来高やビッド・アスク・スプレッドなど、取引に関するデータが必要です。 |
| 3. シナリオの無効化とレビュー | **外部データが必要** | ピラーIVで設定した外部のサインポストを継続的に監視する必要があります。 |
| 4. 定期的なフレームワーク・レビュー | **外部データが必要** | 我々の分析手法自体を改善するため、新しい投資理論や分析ツールに関する外部の学術研究やレポートをインプットする必要があります。 |
