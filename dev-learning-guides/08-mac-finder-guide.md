# Mac Finder 基本操作ガイド

## 0. はじめに

このガイドは **Mac の Finder を使いこなすための自分用リファレンス** です。

- **想定読者**: Windows のエクスプローラーは使えるが、Mac の Finder は不慣れ
- **ゴール**: Finder の基本操作を理解し、ファイル管理を効率的に行えるようになる
- **前提**: 特になし（どのレベルからでも参照可能）

> Windows のエクスプローラーは「PC」「ドキュメント」「ダウンロード」でファイルを管理するツール。
> Mac の Finder はそれに相当するファイル管理アプリで、常に起動している。
> エクスプローラーが「Windows の案内人」なら、Finder は「Mac の案内人」。

---

## 1. Finder とは

Finder は Mac に最初から入っているファイル管理アプリ。Dock（画面下部のアイコン列）の一番左にある **笑顔のアイコン** が Finder。

### Windows エクスプローラーとの主な違い

| 項目 | Windows エクスプローラー | Mac Finder |
|------|----------------------|-----------|
| 起動方法 | タスクバーのフォルダアイコン | Dock の笑顔アイコン（常に起動中） |
| アドレスバー | パスが常に表示される | デフォルトでは非表示（後述の設定で表示可能） |
| ドライブ | C:, D: などドライブレター | `/`（ルート）から始まる単一のツリー |
| 隠しファイル | 「表示」タブで切り替え | `Cmd + Shift + .`（ドット）で切り替え |
| ファイルの切り取り | `Ctrl + X` → `Ctrl + V` | **切り取りはない**。`Cmd + C` → `Cmd + Option + V`（移動） |
| 削除 | `Delete` キー | `Cmd + Delete` |
| パスの区切り | `\`（バックスラッシュ） | `/`（スラッシュ） |

### Mac のフォルダ構造

```
/（ルート）
├── Users/           ← 全ユーザーのホームフォルダ
│   └── あなたの名前/  ← ホームフォルダ（🏠）
│       ├── Desktop/     ← デスクトップ
│       ├── Documents/   ← 書類
│       ├── Downloads/   ← ダウンロード
│       ├── Pictures/    ← ピクチャ
│       ├── Music/       ← ミュージック
│       └── Movies/      ← ムービー
├── Applications/    ← アプリケーション
├── Library/         ← システムライブラリ（通常は触らない）
└── System/          ← macOS のシステムファイル（触らない）
```

> **ホームフォルダ** = Windows の `C:\Users\あなたの名前` に相当。
> ターミナルでは `~`（チルダ）で表される。

## 2. 画面構成を理解する

Finder ウィンドウは以下の構成になっている。

```
┌──────────────────────────────────────────────┐
│ ◉ ◉ ◉  ← → │ 表示切替  │  検索バー        │  ← ツールバー
├──────┬───────────────────────────────────────┤
│      │                                       │
│サイド│  メインエリア                           │
│バー  │ （ファイル・フォルダの一覧）              │
│      │                                       │
│よく使│                                       │
│う項目│                                       │
│      │                                       │
├──────┴───────────────────────────────────────┤
│  パスバー（設定で表示）                        │  ← 下部
│  ステータスバー（設定で表示）                   │
└──────────────────────────────────────────────┘
```

### 各エリアの役割

| エリア | 場所 | 役割 | Windows で例えると |
|-------|------|------|------------------|
| **ツールバー** | 上部 | 戻る・進む・表示切替・検索 | エクスプローラーのリボン・アドレスバー |
| **サイドバー** | 左側 | よく使うフォルダ・ドライブ・タグ | エクスプローラーのナビゲーションペイン |
| **メインエリア** | 中央〜右 | ファイルとフォルダの一覧 | エクスプローラーのファイル一覧 |
| **パスバー** | 下部 | 現在の場所をパスで表示 | エクスプローラーのアドレスバー |
| **ステータスバー** | 最下部 | 項目数・空き容量 | エクスプローラーのステータスバー |

## 3. 表示スタイル

Finder には 4 つの表示スタイルがある。ツールバーのボタンまたはショートカットで切り替える。

| 表示 | ショートカット | 特徴 | 向いている場面 |
|------|-------------|------|-------------|
| **アイコン** | `Cmd + 1` | サムネイルが大きく見える | 画像フォルダの閲覧 |
| **リスト** | `Cmd + 2` | ファイル名・日付・サイズが一覧で見える | 日常のファイル管理（おすすめ） |
| **カラム** | `Cmd + 3` | フォルダ階層を横に展開 | 深いフォルダ構造の探索 |
| **ギャラリー** | `Cmd + 4` | プレビューが大きく表示される | 写真の選別 |

> **おすすめ**: 普段は **リスト表示**（`Cmd + 2`）が最も使いやすい。
> Windows のエクスプローラーの「詳細」表示に相当する。

## 4. 基本操作

### ファイル・フォルダの操作

| やりたいこと | 操作 | Windows との違い |
|------------|------|----------------|
| 開く | ダブルクリック or `Cmd + O` | 同じ |
| 新規フォルダ作成 | `Cmd + Shift + N` | `Ctrl` → `Cmd` |
| コピー | `Cmd + C` → `Cmd + V` | `Ctrl` → `Cmd` |
| **移動** | `Cmd + C` → **`Cmd + Option + V`** | **Windows の「切り取り」とは違う操作** |
| 名前変更 | ファイル選択 → `Enter` | Windows は `F2`。Mac の `Enter` は開くではなくリネーム |
| 削除（ゴミ箱へ） | `Cmd + Delete` | Windows は `Delete` キーのみ |
| 完全削除 | `Cmd + Option + Delete` | Windows は `Shift + Delete` |
| 情報を見る | `Cmd + I` | Windows の「プロパティ」に相当 |
| クイックルック | `Space` キー | ファイルを選択してスペースで即プレビュー。Windows にはない機能 |

> **重要**: Mac では `Enter` キーは「開く」ではなく「名前変更」。
> ファイルを開くのはダブルクリックまたは `Cmd + O`。

### 複数ファイルの選択

| 操作 | 方法 |
|------|------|
| 連続選択 | 最初のファイルをクリック → `Shift` を押しながら最後のファイルをクリック |
| 個別選択 | `Cmd` を押しながらファイルをクリック（Windows の `Ctrl` クリックに相当） |
| 全選択 | `Cmd + A` |

### ドラッグ & ドロップ

| 操作 | 結果 |
|------|------|
| 同じドライブ内でドラッグ | **移動** |
| 別のドライブへドラッグ | **コピー** |
| `Option` を押しながらドラッグ | 強制的に **コピー** |
| `Cmd` を押しながらドラッグ | 強制的に **移動** |

## 5. ナビゲーション

### 基本のナビゲーション

| やりたいこと | 操作 |
|------------|------|
| 一つ上のフォルダへ | `Cmd + ↑` |
| フォルダを開く | `Cmd + ↓` またはダブルクリック |
| 戻る | `Cmd + [` |
| 進む | `Cmd + ]` |
| ホームフォルダへ | `Cmd + Shift + H` |
| デスクトップへ | `Cmd + Shift + D` |
| ダウンロードへ | `Cmd + Option + L` |
| アプリケーションへ | `Cmd + Shift + A` |
| パスを指定して移動 | `Cmd + Shift + G`（パスを直接入力） |

> **`Cmd + Shift + G` は非常に便利**。ターミナルで見たパスをそのまま貼り付けて移動できる。
> 例: `/Users/matsubarakouji/ai-agent` と入力すれば直接そのフォルダを開ける。

### サイドバーの活用

サイドバーには以下のセクションがある。

| セクション | 内容 |
|-----------|------|
| **よく使う項目** | デスクトップ・書類・ダウンロードなど |
| **iCloud** | iCloud Drive のフォルダ |
| **場所** | Mac 本体・外部ドライブ・ネットワーク |
| **タグ** | 色付きタグで分類したファイル |

**サイドバーに自分のフォルダを追加する方法**:
1. 追加したいフォルダを Finder で開く
2. そのフォルダをサイドバーの「よく使う項目」にドラッグ

> プロジェクトフォルダをサイドバーに追加しておくと、一発でアクセスできて便利。

## 6. 検索

### Finder 内検索

1. Finder ウィンドウ右上の検索バーに文字を入力（`Cmd + F`）
2. 「このMac」または「現在のフォルダ」を選択
3. 結果が表示される

### Spotlight 検索（Mac 全体の検索）

| 操作 | 説明 |
|------|------|
| `Cmd + Space` | Spotlight を開く |
| 検索ワード入力 | ファイル名・アプリ名・計算式・辞書などを横断検索 |

> Spotlight はファイル検索だけでなく、**アプリの起動にも使える**。
> 例: `Cmd + Space` → `terminal` と入力 → `Enter` でターミナルが起動。

## 7. おすすめ設定

Finder はデフォルトのままだと情報が少ない。以下の設定を変更すると格段に使いやすくなる。

### パスバーを表示する

メニューバー → **表示** → **パスバーを表示**（`Cmd + Option + P`）

現在のフォルダまでのパスが下部に表示される。パスの各フォルダをダブルクリックで移動もできる。

### ステータスバーを表示する

メニューバー → **表示** → **ステータスバーを表示**（`Cmd + /`）

項目数とディスクの空き容量が表示される。

### タイトルバーにパスを表示する

ターミナルで以下のコマンドを実行:

```bash
defaults write com.apple.finder _FXShowPosixPathInTitle -bool true
killall Finder
```

ウィンドウのタイトルバーに完全なパス（`/Users/...`）が表示されるようになる。

> ターミナルで作業するときに「今 Finder で見ているフォルダのパス」がすぐわかるので非常に便利。

### 拡張子を常に表示する

メニューバー → **Finder** → **設定**（`Cmd + ,`）→ **詳細** タブ → **すべてのファイル名拡張子を表示** にチェック

### 新規ウィンドウのデフォルトフォルダを変更する

メニューバー → **Finder** → **設定**（`Cmd + ,`）→ **一般** タブ → **新規 Finder ウィンドウで次を表示** をホームフォルダに変更

> デフォルトでは「最近の項目」が表示されるが、ホームフォルダの方が使いやすい場合が多い。

### 隠しファイルを表示する

`Cmd + Shift + .`（ドット）で表示 / 非表示を切り替え。

> `.gitignore` や `.claude/` など、開発で使う隠しファイルを確認するときに必要。
> 普段は非表示にしておき、必要なときだけ切り替えるのがおすすめ。

## 9. Finder とターミナルの連携

開発作業では Finder とターミナルを行き来することが多い。便利な連携方法を知っておくと効率が上がる。

### Finder で見ているフォルダをターミナルで開く

1. Finder で対象フォルダを開く
2. フォルダを **ターミナルにドラッグ** すると、パスが自動入力される

または:

1. Finder で対象フォルダを開く
2. メニューバー → **サービス** → **フォルダに新規ターミナルを開く**（要設定※）

> ※ **システム設定** → **キーボード** → **キーボードショートカット** → **サービス** → **ファイルとフォルダ** → 「フォルダに新規ターミナルを開く」を有効化

### ターミナルから Finder を開く

```bash
open .          # 現在のフォルダを Finder で開く
open ~/Desktop  # デスクトップを Finder で開く
open file.txt   # ファイルをデフォルトアプリで開く
```

> `open .` は非常によく使うコマンド。ターミナルで作業中に「このフォルダを Finder で見たい」というときに便利。

### VS Code から Finder を開く

VS Code のエクスプローラーでファイルやフォルダを右クリック → **Reveal in Finder**（Finder で表示）

## 10. タグ機能

Finder にはファイルやフォルダに **色付きタグ** を付ける機能がある。

### タグの付け方

1. ファイルを右クリック
2. カラータグ（赤・オレンジ・黄・緑・青・紫・グレー）をクリック

または:

1. ファイルを選択
2. ツールバーの **タグ** ボタンをクリック
3. 色を選択、または自由なテキストタグを入力

### タグの活用例

| タグの色 | 用途の例 |
|---------|---------|
| 🔴 赤 | 緊急・重要 |
| 🟠 オレンジ | 作業中 |
| 🟢 緑 | 完了 |
| 🔵 青 | 参照資料 |

> タグはサイドバーからフィルタリングできる。フォルダの構造に関係なく、タグで横断的にファイルを探せる。

## 11. 知っておくと便利な Tips

### クイックルック（Space キー）の活用

ファイルを選択して **Space** を押すだけで、アプリを開かずに中身をプレビューできる。

- PDF → 中身が表示される
- 画像 → プレビュー表示
- テキストファイル → 内容が読める
- 動画 → その場で再生

> **複数ファイルを選択して Space** を押すと、矢印キーで順番にプレビューを切り替えられる。

### スタック（デスクトップの自動整理）

デスクトップにファイルが散らかりがちなら:

1. デスクトップの何もない場所を右クリック
2. **スタックを使用** をクリック

ファイルが種類別に自動でグループ化される。

### AirDrop

Mac 同士・iPhone/iPad とファイルをワイヤレスで送受信できる。

1. Finder のサイドバー → **AirDrop** をクリック
2. 近くのデバイスが表示される
3. ファイルをドラッグ & ドロップで送信

### zip 圧縮・展開

| 操作 | 方法 |
|------|------|
| 圧縮 | ファイルを右クリック → **「ファイル名」を圧縮** |
| 展開 | `.zip` ファイルをダブルクリック |

## 12. よくあるトラブルと対処法

### 「このアプリケーションは壊れているため開けません」

インターネットからダウンロードしたアプリで発生することがある。

```bash
xattr -cr /Applications/アプリ名.app
```

ターミナルでこのコマンドを実行すると、検疫属性が解除されて開ける。

### 「.DS_Store」ファイルが邪魔

`.DS_Store` は Finder がフォルダの表示設定を保存するための隠しファイル。Git リポジトリに含めたくない場合:

```bash
echo ".DS_Store" >> .gitignore
```

> このプロジェクトの `.gitignore` にはすでに `.DS_Store` を追加済み。

### ファイルが見つからない

1. **Spotlight** で検索（`Cmd + Space`）
2. Finder で `Cmd + F` → 検索条件を設定
3. ターミナルで `find` コマンドや `mdfind` コマンドを使う

### ゴミ箱を空にできない

```bash
sudo rm -rf ~/.Trash/*
```

> 通常は Finder の **ゴミ箱** → **ゴミ箱を空にする** で対応可能。
> それでも消えないファイルがある場合のみ、ターミナルで対処する。
